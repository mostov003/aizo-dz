# شرح البنية المنظمة للكود

## التحسينات المطبقة:

### 1️⃣ **فصل الإعدادات (config.js)**
- نقل كل المتغيرات الحساسة من الكود إلى ملفات البيئة
- تحميل آمن للإعدادات من `dotenv`
- تحذيرات إذا كانت المتغيرات ناقصة

### 2️⃣ **إدارة متغيرات البيئة**
- **`.env.example`**: قالب يوضح المتغيرات المطلوبة (آمن للتشارك)
- **`.env`**: ملف فعلي بقيم حقيقية (في .gitignore - لا يُشارك أبداً)

### 3️⃣ **فصل المسارات (routes/shipments.js)**
- مسار منفصل لإدارة الشحنات
- Middleware للتحقق من صحة البيانات
- تنظيم أفضل للكود

### 4️⃣ **تحديث الملفات**
```
server.js                    # التحديثات:
├── استيراد dotenv         # تحميل متغيرات البيئة
├── استيراد config.js      # الإعدادات المركزية
└── استيراد routes         # المسارات المنظمة

config.js (جديد)           # إدارة الإعدادات
routes/
└── shipments.js (جديد)    # مسارات ZR للشحن

.env (جديد)                # متغيرات البيئة الفعلية
.env.example (جديد)        # قالب توثيقي
.gitignore (جديد)          # منع تشارك الملفات الحساسة
```

## الاستخدام:

### 📦 تثبيت المتطلبات:
```bash
npm install
```

### 🚀 تشغيل الخادم:
```bash
npm start
```

### 🔧 تكوين البيانات الحساسة:
1. انسخ `.env.example` إلى `.env`
2. أضف قيمك الحقيقية في ملف `.env`
3. لا تشارك ملف `.env` في Git

## المسارات الجديدة:

### إنشاء شحنة:
```
POST /api/shipments/create
Content-Type: application/json

{
  "orderId": "ORD-001",
  "customerName": "أحمد علي",
  "customerAddress": "الجزائر العاصمة",
  "customerPhone": "0555123456",
  "weight": 1.5
}
```

### الرد الناجح:
```json
{
  "success": true,
  "message": "تم شحن الطلب عبر ZR بنجاح",
  "data": { /* زر البيانات من ZR API */ }
}
```

### الأخطاء:
- **400**: بيانات ناقصة أو غير صحيحة
- **500**: خطأ في الخادم أو الاتصال بـ ZR API

## أفضل الممارسات المطبقة:

✅ **أمان**: المفاتيح الحساسة في `.env` وليس في الكود  
✅ **تنظيم**: فصل الإعدادات عن المسارات  
✅ **توثيق**: تعليقات واضحة وملف `.env.example`  
✅ **التحقق**: Middleware للتحقق من صحة البيانات  
✅ **معالجة الأخطاء**: رسائل خطأ واضحة بالعربية  
✅ **قابلية التطوير**: سهولة إضافة مسارات وتحسينات جديدة
